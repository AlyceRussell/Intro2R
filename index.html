RStudio

Throughout this workshop, we will be using RStudio: a free, open source R integrated development environment.

The advantage of RStudio is that it's user friendly, particularly to those who do not have a computer science background (myself included!).

Basic Layout

When you first open RStudio, you will be greeted by three panels:

  • The interactive R console (entire left)
  • Workspace/History (tabbed in upper right)
  • Files/Plots/Packages/Help (tabbed in lower right)

Once you open files, such as R scripts, an editor panel will also open in the top left.

Executing Commands

There are a few ways you can execute commands in RStudio. You can run code from the current line by:

  1. clicking the "Run" key in the top right corner of the editor panel
  2. going to "Code" menu then "Run Line(s)"
  3. hit Ctrl+Enter in Windows or Linux OS, or Command+Enter in Mac OS
  4. hit Ctrl+R in Windows OS

You can also highlight a code chunk and use the above options.

Accessing Help

  • Type '?keyword' in the console panel if the package is loaded. For example, ?sum or ?plot
  • Type '??keyword' in the console panel if the package/function is unknown
  • Type 'help(keyword) in the console panel. For example, help(sum) or help(plot)
  • In RStudio, go to the help menu then R help, type in keywords to see options

The Infamous Working Directory

The working directory is where you import files from and export files to when using R.

To find your current working directory at anytime...

Navigation

  • Use the up and down arrows in the interactive R console (bottom left) to scroll through and even edit previous commands.
  • Use Ctrl+L in Windows (or Alt+88+L in Mac) to clear the console.

R Language Basics

Assignment

One of the most important things you will learn in R is the assignment of a dataset/matrix/vector/value/character to a variable of your choosing. For assignment you can use either "=" or "<-", though I personally prefer to use "<-" for assignment and "=" when you're running a logical argument (more on that later).

R can be thought of as a sophisticated calculated as well as programming environment. For example, you can assign values to variables and then use these in equations you assign to new variables, as follows...

a <- 1
b <- 2
c <- a + b
c

NOTE: R is case sensitive, thus "Hey", "HEY" and "hEY" will be considered as different variables within the R environment. Use CamelCase to write all your variables and avoid confusion!!

R's view on Space

R does not recognise spaces in sentences. The following are equivalent...

a1 <- 2 + 2
a2<-2+2
a1
a2

In saying this, all R variables need to contain no spaces. This is why CamelCase is useful. You can also use other symbols, such as _ or ., in your variable name.

The key is to choose variable names that aren't too laborious when typing, intuitive to the information they contain and aren't too confusing when compared with other variables in your current R environment (so you don't accidently include the wrong variable in your analysis).

In some cases, an assignment needs to be made over a few lines. For this, you can press Enter during the command and it will take you to a new but indented line, as shown in the below plot code.

dev.off()   #resets plotting settings from first image
q <- c(2, 5, 4, 7, 1, 8, 10)
w <- c(1, 2, 3, 4, 5, 6, 7)
plot(q, w, main="Example Plot",
     type="b", col="pink", lwd="5")

Some Final Pointers

Assignment should never be made to a variable that is also a name of a standard function (for example, sum or sd or max).

You can assign the last value or expression to a new variable using the .Last.value function.

z <- .Last.value

Data Types in R

Data in R is not exclusively numeric; there are a number of different types of data.

  • Numeric
    • Integer (2, 5, ...)
    • Double (1.3, -2.56, ...)
    • Complex (a square root of a negative number, i.e. contains imaginary numbers)
    • Logical (0 or 1 as a result from a test)
  • Character ("John", "This is some code", ...)
    • Date/Time (Sep 1, 2012, 24/3/2013, January 5 2009, ...)
  • Factor (SES may have "Low", "Middle", "High"; or Likert on questionnaire "1", "2", "3", "4", "5")

You can test for and even convert the data type using typeof() and as.xxxx(), where "xxxx" can be "numeric", "factor", "double", "integer", "complex", "character", "logical".

Data Conversions

Converting from... Converting to... Result
Integer Double/Complex Number is maintained when back to Integer
Double Integer Decimal is lost
Double Complex Number is maintained when back to Double
Complex Double/Integer Imaginary part is lost
Double/Integer Logical Value TRUE is returned for everything except 0, which is FALSE
Logical Double/Integer Value of 0 for FALSE and 1 for TRUE
Anything Factor Becomes Factor levels of variable
Character Numeric Only if characters are numbers, otherwise returns NA 
a <- c(1, 2, 3, 4, 5)
typeof(a)  #double is default for numbers
b <- as.integer(a)  #converts double into integer
typeof(b)
c <- as.factor(a)  #converts integer into factor
typeof(c)  
levels(c)  #calls levels of a factor

Data Structures in R

There are a number of different data structures in R, which include:

  • Scalers/Variables (0-dimensional)
  • Vectors (1-dimensional)
  • Matrices (2-dimensional)
  • Arrays (n-dimensional)
  • Data frames (Standard in datasets)
  • Lists

You can assign any of these data structures to a variable and perform a number of tasks. I will only introduce you to some basic operations with these structures today.

Vectors

Vectors are a string of values greater than 1, and there are a number of ways to generate vectors.

v1 <- 3  #assign single value, as a scaler/variable example

v6
v7 <- exp(v2)  #assign values from previous variable using a mathematical function

v9 <- paste("Demo_", rep(LETTERS[1:4], each=3), c(1:10), sep="")  #sequentially pastes numerous vectors together

Matrices

Matrices are 2-dimensional vectors. Below are some examples of generating matrices in R.

a <- matrix(data=seq(1, 42, length=21), ncol=3)  #creates matrix, 3 columns
a
b <- matrix(data=seq(1, 42, length=21), nrow=7, ncol=3, byrow=TRUE)  #fills matrix by row instead of column
b
c <- matrix(seq(1, 42, length=21), 7, 3, TRUE)  #same as previous code with option names omitted - must be in right order
c
d <- diag(4)  #creates 4x4 identity matrix
d

There are many ways of manipulating matrices. 

a[1,]  #displays first row of matrix 
a[,1]  #displays first column of matrix
e <- rbind(a[2,], a[6,])  #row binds the second and sixth rows of matrix together
e
f <- cbind(e, c(2, 3))  #column binds matrix e with new vector
f
g <- matrix(1:4, 2)  #creates matrix with 2 rows
h <- matrix(5:8, 2)
g + h  #element wise addition
g * h  #element wise multiplication
g %*% h  #matrix multiplication
g %o% h  #outer product
t(c)  #transpose matrix
solve(g)  #inverse of matrix (must be a square matrix)
diag(h)  #shows diagonal elements of matrix (unlike diag(4) in above example, which creates a 4x4 identity matrix)
nrow(g)  #number of rows
ncol(h)  #number of columns
rowMeans(a)  
rowSums(a[,2:3])  #sums rows over columns 2 and 3. 
colMeans(a)
colSums(a)

NOTE: The basic structure for indexing a matrix/data frame is variable[row, column]. If you want all the rows but are only interested in indexing some of the columns (say, to sum questionnaire results), you leave the row part blank and only tell R which columns you are interested in. You can use the concatenate if need be to select columns not adjacent to each other: e.g. rowSums(x[,c(1, 3:5)])

Data Frames

When you are manipulating your imported datasets, they are more often than not structured as a data frame. In data frames, the observations are the rows and the variables are the columns. You can call out individual/groups of row/s column/s to perform a variety of tasks, which we will look at shortly. 

ds1 <- data.frame(FirstName=c("Steve", "Amy", "John"), LastName=c("Johnson", "O'Neil", "Smith"), 
                  Age=c(20, 23, 19), Gender=c("M", "F", "M"), CourseID=c("K05", "M22", "E04"), 
                  WAM=c(67, 79, 62))  #creates a data frame
ds1
names(ds1)  #checking variable (column) names
FullName <- paste(ds1$FirstName, ds1$LastName, sep=" ")  #create new FullName variable
ds1 <- cbind(ds1, FullName)  #column bind new variable to data frame
names(ds1)  #checking data frame to see the newly attached FullName variable
names(ds1)[6] <- "WeightedAverage"  #changes name of 6th variable in the data frame
nrow(ds1)  #counts number of rows. Use ncol(ds1) for number of columns. 
dim(ds1)  #gives row and then column count for data frame
mean(ds1$WeightedAverage)  #gives mean weighted average mark between students
ds1[1,6] = 70  #this will index to the individual in row 1's WAM result and update it to 70
ds1

NOTE: When coding a data frame, the columns all have to have the same number of observations. Missing values can be denoted with NA.

We will look at some additional tricks with data frames at the end.

Lists

Lists can store information from a number of different vectors of unequal length and different data types. The output of many statistical tests in R are stored in lists and individual parts of the test results can be called out in the same fashion you call out individual variables in data frames (e.g. ds1$FullName). I will not go into list generation today but you can look it up using ?list in the interactive console (bottom left).

Working with Missing Values

The R function is.na() can be used to see whether there are any missing values in a dataset/matrix/vector/variable.

r <- c(2, NA, 4, 6, 8, 10)
is.na(r)

NOTE: R reads missing values as NA, which is an indication of missingness.

You can replace these in functions, for example, using an imputed mean or some other suitable value, or simply remove rows/columns of data that contain missing values (not recommended in the statistics community). 

m <- as.data.frame(matrix(c(2:6, NA, 15:16, NA, 17, 22, 10), ncol=3))
m
is.na(m) #will return TRUE or FALSE for each data point
length(m[is.na(m)]==TRUE)  #counts missing values in dataset using a logical argument for the above function
length(m$v1[is.na(m)]==TRUE)  #counts missing values in the first column
mean(m)  #calculates mean of all values, won't work if missing values exist
mean(as.matrix(m), na.rm=TRUE)  #omits missing values from calculation of the mean

m_mean <- mean(as.matrix(m), na.rm=TRUE)  #puts calculated mean for non-missing values into a new variable

m <- as.matrix(m)
for (i in 1:length(m)){   #length() used to find number of values in the matrix
  if (is.na(m[i])){
    m[i] = m_mean      #simple loop that adds imputed mean to missing data points
  } else {            
    m[i] = m[i]
  }
}
m

As you may have noticed in the last bit of code, I changed a matrix into a data frame for some tasks then back to a matrix. You can do this using the as.xxxx function, where the xxxx can be matrix, data.frame, etc... I did this because certain functions work better in either format. Knowing which data structure to use for a given task will come with practise.
